# TRPGDice

## これは何?

TRPGでよく使う`"3d6"`みたいな表記の乱数を扱うことができるCLIです。

`"1d6 * 2 - 6"`みたいな表記にも対応しています。

> [!WARNING]
>
> 処理がわりかし遅いのと、メモリアロケートがかなり多いのでゲーム目的には使わないほうが良いです。
> ゲーム内ダイスロールを実装する際は、文字列ベースではなく別のアプローチを取ったほうが良いと思われます。

## 要件

少なくとも[.NET Runtime 10](https://dotnet.microsoft.com/ja-jp/download/dotnet/10.0)が必要です。

## 使い方

1. ソースコードを時前でビルドする OR GitHubからダウンロードすることで実行ファイルを入手する
2. 実行ファイルにPATHを通す
3. `dice "3d6"`のように実行する

## 例

```shell
dice 1d100
>> 87

dice "3d6 + 2"
>> 14

dice "1d6 * 2 - 6"
-2
```

## 実装方針

### `ndm`部の処理

`"3d6"`のように、ダイス表記になっている部分は正規表現で取り出し、乱数の結果を置換する方法を取りました。
置換結果に関数ポインタ・デリゲート等を使用できるRegex/Replaceが実装されている言語であれば、同様のアプローチで実装可能です。

Discord Botやブラウザアプリ用に自力で実装したい場合は、本実装が参考になるかもしれません。

具体的な計算としては、`d`の前の数字を`count`、後の数字を`surface`として取り出し、以下のように繰り返し加算しました。

```cs
for (int i = 0; i < count; i++) {
    sum += rand.Next(1, 1 + surface);
}
```

### 数式全体の処理

数式全体の処理は[xFunc.Maths](https://www.nuget.org/packages/xFunc.Maths)ライブラリの力をお借りしました。

xFuncに処理を投げる時点でダイス表記は全て定数に置換されているため、問題無く処理することができます。
